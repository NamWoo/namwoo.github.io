스스로가 꽤 진전이 있다고 생각했는데 아직도 전처리 중……ㅡㅡ
대충 90만개 행 x 85개 열 데이터 셋을 이리저리 뜯어보는 중에

결측치에 대하여

과정

  1. 각각의 열마다 또 각각 따로 정의된 missing values 가 있어 모두 따로 정의했다.
    1. 이건 따로 csv로 정의되어 있어 비교대조 진행
    2. 그래서 빈칸이든 missing values든 error values든 모두 np.nan 로 변환
  2. 행 별로 기준 결측치 퍼센트 15% 부터 30% 까지 나열
    1. 18%부터 출력되는 행이 6개 확인
    2. 한 행에 결측치 18% 이상인 행 drop
  3. 열 별로 기준 결측치 확인

여기서 의문
행은 카테고리로 정의된 string 값이라, 그러니까 사람의 특징을 나열할 때 나이, 키, 몸무게 같은 특징을 나타내는데 열은 그냥 index도 아니고 그냥 내열된 값이다. 90만번까지 있는 이 열 중에 몇 개를 날려버려도 상관없는 것인가?

  1. 어쨌든 열 별로 기준 결측치 퍼센트로 확인
    1. 결측치가 40% 이상인 열 10만개 열 총 10% drop

잘하고 있는 짓인거? 전체 데이터의 10%면 어마어마한건데…. PCA나 fastICA 진행시 차이가 있을까? 이건 결과값가지고 비교해볼 수 밖에..

  1. 0 또는 1, Yes or No, Y, or N, O or X 등 과 같은 이진값들 확인 후 0,1로 통일.
    1. 근데 어떤 행에는 O, W 이런 이진값이 있었는데..?? 어느게 1? 어느게 0? 그냥 바꿔 내맘대로 바꿈
    2. 1과 0 그리고 1.0과 0.0, int or float
      1. 데이터들이 그냥 정수 인데 .0 이 붙어있는 실수화 되어있다. 소수점 이하 수치가 있으면 고민하지도 않았음.
      2. 이런 int와 float 차이는 어떻게?
      3. 싹 int로 바꿔 버리고 싶었는데 안에 np.nan 있기에 int 불가
      4. 그냥 싹 float로 바꿔? 아님 그냥 나둬?
      5. 일단 놔두고 나중에 비교해보기?
    3. 그냥 np.nan 빼고 모두 int로 바꿔버림

잘… 하는거 맞겠…지?

화남

csv 단일 파일만 150MB. 데이터가 이 정도면 크지 않다고 생각하는데 .copy() 수 단계 거치고 솎아내고 걸러내고 따로 뽑아비교하고.. 점점 커져가다보니 Jupyter가 밀리기 시작함. ipython 말고 그냥 일반 python으로 하고 싶어도 실제 그래프를 보면서 데이터를 비교하고 대조하고 그리고 글도 써내려가고 싶은데…

그리고 클라우드 기반 ipython 사용중이라 리눅스 가상환경 설정도 맞춰줘야함.. import 오류 떄문에 한 고생.. 작성한 cells 모두 실행해놓아도 환경설정 오류나면 import 오류나면서 싹 다시 해야함….. 또 엄청 기다림… 오늘도 도대체 코딩한 시간보다 기다린 시간이 더 많음…. 말도 안돼…

마지막엔 잘 되고 있었는데 끝에가서 크롬 뒤로가기 실행돼서 싹 초기화… 열받아서 드러누음. ㅡㅡ
